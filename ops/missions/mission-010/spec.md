## North star

This run is not about inventing another UI language.

This run is about proving that the canonical agent card can evolve into a more premium, more legible, more alive operator artifact while preserving:

- one shared identity grammar
- one shared status grammar
- one shared provenance grammar
- one bounded child-work pattern

## Non-negotiable rules

- Preserve one canonical card contract.
- No surface-specific card species.
- Parent agent remains the authority face.
- Child units stay attached, bounded, and clearly non-sovereign.
- No telemetry inflation, fake freshness, or surface-specific status drift.
- The result must stay rollout-safe and implementation-ready rather than speculative art direction.

## Required deliverables

1. canonical baseball-card VNext spec
2. shared-vs-variable telemetry matrix for major surfaces
3. subagent affordance and hover-bubble interaction contract
4. component architecture recommendation
5. safe rollout / migration plan

## Acceptance criteria

1. One incubated idea completes flesh / stress / value salons in live mode.
2. The idea is promoted into DEV staging with PRD and dev pack artifacts.
3. The resulting Alpha job is admitted, delegated, and fanned out.
4. All three Alpha seat tasks reach completed with proof backbriefs.
5. President-A accepts all required proofs.
6. The publish-ready artifact bundle is filed.
7. The job archives and seals cleanly.
8. The deliverable clearly improves operator truth for identity, provenance, activity, and child-work visibility.

## Fail-fast blockers

1. A second card species is proposed.
2. Subagents are treated like peer seats or new sovereign cards.
3. Status meanings drift between surfaces.
4. The deliverable relies on fake live telemetry to appear premium.
5. Closeout cannot point to concrete artifacts that make implementation safe.
