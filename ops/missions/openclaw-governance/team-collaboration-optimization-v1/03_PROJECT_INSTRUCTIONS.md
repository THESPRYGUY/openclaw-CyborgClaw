# 03_PROJECT_INSTRUCTIONS

## Purpose

Define always-on operating rules for Codex and the operator during this mission.

## Operating posture

- evidence first
- no guessing
- fail loud on ambiguity
- smallest bounded change first
- branch-first safety
- durable evidence over narration

## Required reporting pattern

Every substantive update should include:

1. traffic light
2. verified current state
3. target architecture
4. 10-year-old explanation
5. main answer
6. one next action
7. what good looks like

## State language

Use:

- VERIFIED
- LIKELY
- UNKNOWN
- TO VERIFY

## Prompting discipline

- choose the route using `PROMPT_GUIDE/02_WORKLOAD_AND_ROUTE_MATRIX.md` and `PROMPT_GUIDE/09_OPERATOR_DECISION_TREE.md`
- prefer the smallest matching template from `PROMPT_GUIDE/04_PROMPT_TEMPLATES.md`
- use `PROMPT_GUIDE/07_STARTUP_AND_REANCHOR_PROMPTS.md` when restarting or re-scoping a noisy session

## Closeout rule

A mission is not done until deliverables, validation proof, and handoff artifacts all exist.
