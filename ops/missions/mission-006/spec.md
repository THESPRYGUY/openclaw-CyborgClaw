# Mission 006 — Overnight Live-Fire Hardening

Goal:
Run a 10-loop recursive self-improvement canary against CyborgClaw so the operator-facing MVP surfaces behave truthfully under live use.

## Primary targets

- Agent Design Studio
- Strike Team Design Studio
- Workforce mission room / agent foyer telemetry seams
- STUD boot sequence and launch truth
- Mission docs, receipts, and paperwork rails

## Campaign north star

This mission is not about adding more features.

This mission is about proving that CyborgClaw can:

1. design a specialist seat truthfully
2. compose a governed strike team truthfully
3. publish artifacts without implying runtime activation
4. boot a mission through STUD with real PASS / WARN / BLOCK gating
5. keep telemetry, docs rails, and agent cards aligned across surfaces
6. improve itself through RSI without becoming a second authority

## Non-negotiable rules

- Runtime authority stays downstream.
- RSI is draft-producing and eval-first only.
- If telemetry is uncertain, label it as stale, partial, or projected.
- One canonical agent baseball-card contract must hold across the foyer and mission room.
- Mission paperwork and receipts must stay visible and truthful.
- The smallest safe fix wins over broad refactors.

## Loop map

### Loop 1 — Live telemetry truth

- Target: agent foyer, workforce mission room, team selector
- Goal: remove stale or misleading posture summaries

### Loop 2 — Canonical card enforcement

- Target: foyer tiles, mission room tiles, shared card grammar
- Goal: collapse drift back onto one canonical CyborgClaw card contract

### Loop 3 — Agent Design Studio readiness truth

- Target: seat readiness, first-contact UI, next action
- Goal: remove false confidence and reduce first-read noise

### Loop 4 — Strike Team Design Studio formation truth

- Target: commander/core/bench board, team blockers, seat-source handoff
- Goal: keep the board formation-first and truthful

### Loop 5 — STUD boot sequence hardening

- Target: preflight phases and launch truth
- Goal: add the missing paperwork-path check and keep false-green launches blocked

### Loop 6 — RSI default behavior

- Target: draft-only, eval-first improvement posture
- Goal: confirm the existing canary doctrine still holds under the current UI flow

### Loop 7 — Publish / runtime separation

- Target: lifecycle wording and operator interpretation
- Goal: keep publish, runtime accepted, and effective now unmistakably separate

### Loop 8 — Paperwork / artifacts / receipts

- Target: docs rail, dispatch ledger, proof index, closeout posture
- Goal: make docs statuses honest about what is live versus merely waiting

### Loop 9 — Degraded / failure modes

- Target: stale seats, staged teams, empty teams, partial payloads
- Goal: fail calmly and legibly

### Loop 10 — Full end-to-end canary

- Target: seat design, team composition, STUD preflight, route checks, and receipts
- Goal: prove the MVP loop is close enough to trust for continued canary rollout

## Success bar

Call Mission 006 successful only if:

- team and seat telemetry is truthful and acceptably fresh
- the canonical agent card grammar is enforced across the main surfaces
- Agent Design Studio feels seat-first and honest
- Strike Team Design Studio feels formation-first and honest
- STUD blocks false-green launches
- publish / runtime separation remains unmistakable
- paperwork rails are truthful enough to trust
- the operator leaves with a calm, understandable next move
